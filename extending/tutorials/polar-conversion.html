Using the Polar Conversion Helper¶

Here is a list of tutorials and resources that this tutorial assumes you are already familiar with.

  • Polar Coordinate System
  • Cartesian Coordinate System
  • Writing a Domain Implementation

There are occasions when writing a RealDomain implementation, say a Translation transform where the “interesting” code is contained within the _get_x and _get_y methods and all you need is a conversion from cartesian to polar coordinates for the other methods.

This is where the PolarConversion helper class comes in, it provides implementations of _get_r and _get_t that convert the \(x\) and \(y\) values generated by your _get_x and _get_y methods into their polar equivalents.

This tutorial walks you through the process of using this helper to implement an example translation domain transform.

Setup¶

As with any other domain transformation is starts with us creating a class and subclassing RealDomainTransform. However we also include PolarConversion in our list of base classes.

from stylo.domain.helpers import PolarConversion
from stylo.domain.transform import RealDomainTransform

class Translation(PolarConversion, RealDomainTransform):
    pass

Important

The order in which these classes are included is very important. PolarConversion must come before RealDomainTransform

If we were to try and create an instance of our class right now we will get an error (as expected since RealDomainTransform is an abstract class)

>>> translate = Translation()
Traceback (most recent call last):
   ...
TypeError: Can't instantiate abstract class Translation with abstract methods _get_x, _get_y, _repr

But notice that _get_r and _get_t don’t appear in the error message, this is due to them being implemented by the PolarConversion helper.

Implementation¶

If you’ve implemented a domain transform before, the rest should be familiar to you it’s a case of implementing the remaining methods.

  • __init__: We still need to take input from the user.
  • _repr: We need a textual representation of the object which will be used by the __repr__ method in the base class.
  • _get_x: A method to generate the \(x\) values, here we will implement the horizontal shift
  • _get_y: A method to generate the \(y\) values, here we will implement the vertical shift.

__init__¶

The __init__ method is pretty straightforward, it’s a case of taking a couple of input values from the user to represent the translation. Just don’t forget to call the __init__ method on the base class!

def __init__(self, domain, dx, dy):
    super().__init__(domain)

    self.dx = dx
    self.dy = dy

_repr¶

Again nothing too interesting here, we need to return a string representing our translation to the user.

def _repr(self):
   return "Translation: ({}, {})".format(self.dx, self.dy)

The reason domain transforms like RealDomainTransform ask you to implement _repr and not __repr__ is that the base class implements __repr__ in a way that also displays that the transformation is acting upon.

_get_x¶

Here we implement the first half of the transform, the horizontal shift. First we need to get the \(x\) values from the domain we are acting on and then shift them by the value given to us by the user.

def _get_x(self):

    xs = self.domain.x

    def mk_xs(width, height):
        return xs(width, height) - self.dx

    return mk_xs

Notice that we need to take away the value given by the user, this is so that when they give a positive number the shape they draw is also moved in the positive direction.

_get_y¶

Here we implement the second hald of the transform, the vertical shift. First we need to get the \(y\) values from the domain we are acting on and then shift them by the value given by the user.

def _get_y(self):

    ys = self.domain.y

    def mk_ys(width, height):
        return ys(width, height) - self.dy

    return mk_ys

Notice that we need to take away the value given by the user, this is so that when they give a positive number the shape they draw is also moved in the positive direction.

Bringing It All Together¶

Here is the complete definition

>>> translated = Translation(UnitSquare(), 1, 0)
>>> translated
Translation: (1, 0)
  UnitSquare: [0, 1] x [0, 1]

>>> translated.r(4, 4)
array([[1.41421356, 1.20185043, 1.05409255, 1.        ],
       [1.20185043, 0.94280904, 0.74535599, 0.66666667],
       [1.05409255, 0.74535599, 0.47140452, 0.33333333],
       [1.        , 0.66666667, 0.33333333, 0.        ]])
