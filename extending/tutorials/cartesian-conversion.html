Using the CartesianConversion Helper¶

Here is a list of tutorials and resources that this tutorial assumes you are already familiar with.

  • Polar Coordinate System
  • Cartesian Coordinate System
  • Writing a Domain Implementation

There are occasions when writing a RealDomain implementation, say a Rotation transform where all the “interesting” code is contained within the _get_r and _get_t methods and all you need is a conversion from polar to cartesian coordinates.

This is where the CartesianConversion helper class comes in, it provides implementations of the _get_x and _get_y methods to convert them from the _get_r and _get_t methods that you provide.

This tutorial walks you through the process of using this helper to implement an example rotation domain transformation.

Setup¶

As with any other domain transformation it starts with us creating a class and subclassing RealDomainTransform. However in this case we also include CartesianConversion in our list of base classes

from stylo.domain.transform import RealDomainTransform
from stylo.domain.helpers import CartesianConversion


class Rotation(CartesianConversion, RealDomainTransform):
    pass

Important

The order in which these classes are included is very important. CartesianConversion must come before the RealDomainTransform

If we were to try and create an instance of our class now we will get an error (as is expected since RealDomainTransform is an abstract class).

>>> rotate = Rotation()
Traceback (most recent call last):
   ...
TypeError: Can't instantiate abstract class Rotation with abstract methods  _get_r, _get_t, _repr

But notice that _get_x and _get_y don’t appear in the error message, this is due to them being implemented by the CartesianConversion helper.

Implementation¶

If you’ve implemented a domain transform before, the rest should be familiar to you, it’s just a case of implementing the remaining methods:

  • __init__: We need to take an angle as input from the user
  • _repr: We need a textual representation of the object, this is used by the __repr__ method in the RealDomainTransform base class
  • _get_r: A method to generate the \(r\) values
  • _get_t: A method to generate the \(\theta\) values and here is where the actual rotation will be implemented

__init__¶

The __init__ method is pretty straightforward it’s just a matter of taking a value from the user to represent an angle. Just don’t forget to call the __init__ method on the base class!

def __init__(self, domain, angle):
    super().__init__(domain)

    self.angle = angle

_repr¶

Again nothing too groundbreaking here, just a case of returning some meaningful string to represent our rotation to the user.

def _repr(self):
    return "Rotation: {:.2f}rad".format(self.angle)

The reason domain transforms like RealDomainTransform ask you to implement _repr and not __repr__ is that the base class implements __repr__ in a way that also displays the domain that the transformation is acting upon.

_get_r¶

The \(r\) values don’t change when rotated so all we need to do is return the \(r\) values from the base domain.

def _get_r(self):
    return self.domain.r

_get_t¶

Here is the only non trivial method on this class. We need to get the base domain to generate its \(\theta\) values for us which we then take away the user’s angle which implements the rotation

def _get_t(self):

    ts = self.domain.t

    def mk_ts(width, height):
        return ts(width, height) - self.angle

    return mk_ts

Bringing It All Together¶

Here is the complete class definition

from math import pi

from stylo.domain import UnitSquare
from stylo.domain.helpers import CartesianConversion
from stylo.domain.transform import RealDomainTransform


class Rotation(CartesianConversion, RealDomainTransform):

    def __init__(self, domain, angle):
        super().__init__(domain)

        self.angle = angle

    def _repr(self):
        return "Rotation: {:.2f}rad".format(self.angle)

    def _get_r(self):
        return self.domain.r

    def _get_t(self):

        ts = self.domain.t

        def mk_ts(width, height):
            return ts(width, height) - self.angle

        return mk_ts

Which we can now use to apply a rotation to some domain

>>> rotated = Rotation(UnitSquare(), pi/4)
>>> rotated
Rotation: 0.79rad
  UnitSquare: [0, 1] x [0, 1]

>>> rotated.x(4, 4)
