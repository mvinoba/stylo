Code Style¶

One of the checks your code must pass in order to make Travis happy is the linting test. This test checks two things:

  1. That the black code formatter wouldn’t make any changes to your code’s formatting.
  2. That the flake8 linter doesn’t see any issues with your code.

You can easily run the same tests that Travis would by running the following command

$ pipenv run tox -e lint

Which will run the two tools against the code in both the stylo/ directory and the tests/ directory.
