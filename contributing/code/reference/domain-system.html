The Domain System¶

A Domain is responsible for converting some abstract notion of a mathematical space into a discrete representation. Since stylo is about creating images this representation is typically a 2D grid of numbers.

This document details the machinery behind the Domain system. More specifically how the define_domain and define_domain_transform functions work. It assumes you are already familiar with how the user interacts with Domain objects and is written for someone who wishes to understand how they work behind the scenes.

This module contains all the code that is required to automatically write both the definition of a new Domain type and the test case that users can use to verify that their implementations of said domains adhere to the expected interface.

stylo.domain._factory.domain_getitem(self, key)[source]¶

Enable the domain[…] syntax for domains.

Domains support the ability to return a function in (width, height) of multiple parameters, for example for a RealDomain you might want a function in both the \(x\) and \(y\) coordinates in which case you would write

>>> values = domain['xy']
>>> values(N, N)
(array([[x0, x1, ..., xN],
        [x0, x1, ..., XN],
        ...
        [x0, x1, ..., xN]]),
 array([[y0, y0, ..., y0],
        [y1, y1, ..., y1],
        ...,
        [yN, yN, ..., yN]]))

Note

The string argument 'xy' only works in the case of single letter parameters. For multi-character parameters such as pt this argument must be some other iterable such as a tuple e.g. ('x', 'pt').

This function constructs the implementation for the __getitem__ method that will be added to new Domain definitions

Parameters:
  • self (object) – The standard python instance argument for methods
  • key (iterable) – The input that is given to the getitem syntax e.g. 'xy'
